MI Brews breweries & neighborhood map

Displays data from Foursquare api and google maps api.

ROLE WEB DESIGNER / DEVELOPER
DATE APRIL 2016

MyBrews! It's intended as a challenge to put together a website in react, using multiple APIs. This includes making function calls to Google Maps and other location-based services like Foursquare. The app allows you to filter the list down as you type. In return, it displays the locations on the map. Heck, the icons even bounce like you expect them to. There's also extra info pulled from the Foursquare API where available. This includes website images, ratings, and phone numbers. The app is responsive, too! It resizes using CSS Flexbox and media queries.

Additional credits
  • Udacity
  • Google maps Api
  • Foursquare API

01 USER INTERFACE

MY BREWS UI

A single page application featuring a map of a neighborhood. Including highlighted locations, third-party data about those locations and various ways to browse the content.

I spent some time on the design on this project since it seemed scaleable and easily replicatable with other subjects. React Router was used to allow the use to vistit each location while remaining a single page app.

MyReads : Bookshelf App

Select and categorize books as you have read, are currently reading, or want to read.

ROLE DEVELOPER
DATE APRIL 2017

My Reads: The foremost objective of this project was to design and develop a web app where users can search and categorize books into categories.

Additional credits
  • Udacity

01 USER INTERFACE

My Reads UI

Through this project, a user can create their own personal library. The user can further categorize the books as "Currently Reading", "Read" or as "Want to Read". These books can be organized in different Shelves.

02 TECHNOLOGIES

My Reads: STACK

This application uses ReactJs, React-router, and react-router-dom, as a single page application. To simplify development, a provided JSON BooksAPI.js contains methods to perform necessary operations on the backend: [getAll, update, search]

03 FUNCTIONALITY

THE Features

  • Each Book in a shelf has a drop-down that can be used to move the books to different shelves.

  • New books can be searched by clicking on the "+" button on the main page of the application.

  • On clicking the "+" button, the user is redirected to a new page and the URL is changed from "/" to "/search".

  • The search page has a text input that may be used to find books. As the value of the text input changes, the books that match that query are displayed on the page, along with a control that lets the user add the book to the library. The search page also has a back arrow button to take the user back to the main page. The Route and Link components from "react-router-dom" have been used to handle the routing between the main page and the search page.

  • Necessary checks have been implemented to handle cases where the book's thumbnail or author is empty, so that the app doesn't crash or throw errors. Functionality to move a book between various shelves has been implemented on both the main and the search page. It is also ensured that a book on a bookshelf, has the same state on both the main application page and the search

RESTAURANT REVIEWS PWA

Restaurants of New York

ROLE WEB DESIGNER
DATE JANUARY 2017

RESTAURANT REVIEWS: For this projects, I incrementally converted a static webpage with many problems, lacks accessibility, and converted the design to be responsive on different sized displays and accessible for screen reader use. I also added a service worker to begin the process of creating a seamless offline experience for your users.

Live Demo: restaurant-reviews

Additional Credits
  • Grow With Google

01 USER INTERFACE

RESTAURANT REVIEWS UI

Users of any device including screenreaders can browse restaurants of New York.

Users can filter restuarants by neighborhood and by cuisine, displaying an image as well as their corresponding map marker on a map.

Users can view a details page for each restaurant with reviews.

02 TECHNOLOGIES

Restaurant Reviews Stack

Most of the code in this project has been written to the ES6 specs for compatibility with modern web browsers and future proofing maintainability.

I used Flexbox to achieve a multidevice responsive design.

03 FUNCTIONALITY

RESTAURANT REVIEWS FEATURES

For the Map I used MapBox. It's free, easy to use, and has a nice design.

Through utilzing a service worker, users can enjoy a seamless offline experience.

This project uses best practices to achieve accessibility among user who use screen readers and other similar aides. It achieved a lighthouse score of 100.

MEMORY GAME

Simple card fip memory game.

ROLE WEB DESIGNER
DATE MAY 2017

This project was made as part of Udacity's Front End Nano Degree Certification.

Live Demo: MEMORY GAME

Additional Credits
  • Udacity

01 USER INTERFACE

MEMORY GAME FEATURES

The User can click to open 2 cards at a time if there is match the cards will stay open and a star will be added, and a color change and sound will indicate the success. If there is no match a star will be taken away and a failed color change indicator. After the user matches all of the cards, a pop-up will appear with the amount of turns taken, stars earned, and the amount of time it took. There is option to play again in the pop-up.

02 TECHNOLOGIES

MEMORY GAME STACK

I used 2 royalty free sound effects from freesound.org, HTML5, CSS, and JavaScript.
